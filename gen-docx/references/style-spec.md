# Word 文档格式规范（style-spec）

> 本规范基于 `simple.docx` 提取，面向 `python-docx` 库生成 Word 文档时使用。

---

## 1. 页面设置

### 纸张与页边距

| 项目 | 数值 |
|------|------|
| 纸张大小 | A4 (21 cm x 29.7 cm) |
| 上边距 | 2.54 cm (1 英寸) |
| 下边距 | 2.54 cm (1 英寸) |
| 左边距 | 3.18 cm (1.25 英寸) |
| 右边距 | 3.18 cm (1.25 英寸) |
| 页眉距边界 | 851 twips (约 1.5 cm) |
| 页脚距边界 | 992 twips (约 1.75 cm) |

**文档网格**：`python-docx` 对网格支持有限，部分段落显式设置 `snap_to_grid=False`。

---

## 2. 字体与字号

### 实际使用字体

- 中文：**仿宋**（优先）或方正仿宋简体
- 西文/数字：跟随中文字体设置
- 页脚：继承默认，9 pt

### 字号规则

| 段落类型 | 字号 |
|---------|------|
| 主标题 | **22 pt** |
| 一级标题 / 二级标题 / 正文 / 列表项 | **16 pt** |
| 页脚 | **9 pt** |

---

## 3. 段落样式规范

### 主标题

| 属性 | 设置 |
|------|------|
| 对齐方式 | 居中 |
| 字号 | 22 pt |
| 加粗 | 是 |
| 首行缩进 | 无 |
| 行距 | 默认（单倍行距） |

### 一级标题

| 属性 | 设置 |
|------|------|
| 对齐方式 | 两端对齐 |
| 字体 | 仿宋 |
| 字号 | 16 pt |
| 加粗 | 是 |
| 首行缩进 | 2 字符 (640-643 twips) |

编号：文本编号（如"一、""二、"），直接写入段落文本，不使用自动编号。

### 二级标题

| 属性 | 设置 |
|------|------|
| 对齐方式 | 两端对齐 |
| 字体 | 仿宋 |
| 字号 | 16 pt |
| 加粗 | 是 |
| 首行缩进 | 无（使用列表编号时）或 2 字符 |

编号：文本编号（如"（一）""（二）"）或自动编号（numId 指定）。

### 正文段落

| 属性 | 设置 |
|------|------|
| 对齐方式 | 两端对齐 |
| 字体 | 仿宋 |
| 字号 | 16 pt |
| 加粗 | 否 |
| 首行缩进 | 2 字符 (约 640 twips) |
| 行距 | 默认（单倍行距） |

特殊正文段落（部分）：
- 固定行距 28 pt (560 twips)
- 显式设置 `snapToGrid=False`
- 识别特征：`<w:spacing w:line="560" w:lineRule="exact"/>`

### 列表项

| 属性 | 设置 |
|------|------|
| 对齐方式 | 两端对齐 |
| 字体 | 仿宋 |
| 字号 | 16 pt |
| 加粗 | 否（内容项）或 是（小标题） |
| 首行缩进 | 2 字符 |

### 空白分隔段落

内容为单个空格或空，字体/字号/行距继承默认。

---

## 4. 特殊格式

### 加粗规则

- 主标题、一级标题、二级标题：全部加粗
- 正文：部分关键词或短语加粗

### 黄色高亮

高亮内容通常为待填充数据或需特别注意的内容。

```python
run.font.highlight_color = WD_COLOR_INDEX.YELLOW
```

### 缩进规则

- 首行缩进：大多数段落 2 字符 (640-643 twips)
- 无缩进：标题（居中）、部分列表项
- 左缩进：部分列表项可能有 `w:left` 属性

---

## 5. 编号规则

| 编号类型 | 示例 | 实现方式 |
|---------|------|---------|
| 中文数字 | 一、二、三、 | 文本编号 |
| 中文括号 | （一）（二）（三） | 文本编号或自动编号 |
| 阿拉伯数字 | 1、2、3、 | 文本编号或自动编号 |
| 项目符号 | • | 自动编号（少见） |

**层级**：一级（一、二、）→ 二级（（一）（二）） → 三级（1、2、）

**建议**：优先使用文本编号，如需自动编号需配置 `numbering.xml`。

---

## 6. 页眉页脚

### 页眉

- 页眉距页面顶部 851 twips (约 1.5 cm)
- 具体内容需根据实际需求设置

### 页脚

- 页脚距页面底部 992 twips (约 1.75 cm)
- 字号：9 pt
- 对齐：左对齐（默认）
- 制表位：居中 (4153 twips)、右对齐 (8306 twips)

---

## 7. 字体回退

**优先级**：段落/文字级显式指定 > 样式继承 > 主题字体 > 系统默认

- 中文字符：`w:eastAsia` 指定（如"仿宋"）
- 西文/数字：`w:ascii` 和 `w:hAnsi`
- 缺失回退：宋体 > Calibri

---

## 8. 降级策略

| 原始格式 | 降级方案 |
|---------|---------|
| 固定行距 28pt | 使用单倍行距或 1.5 倍行距 |
| 方正仿宋简体 | 使用仿宋 |
| 自动编号 | 使用文本编号 |
| 文档网格对齐 | 关闭网格对齐 |
| 主题字体映射 | 直接指定字体名称 |
| WPS 特有属性 | 忽略 |

## 10. python-docx 概念映射

| 规范术语 | python-docx 对应 |
|---------|------------------|
| 页面大小 | `section.page_height`, `section.page_width` |
| 页边距 | `section.top_margin`, `section.bottom_margin`, `section.left_margin`, `section.right_margin` |
| 段落对齐 | `paragraph.alignment` |
| 首行缩进 | `paragraph.paragraph_format.first_line_indent` |
| 字体名称 | `run.font.name` |
| 字号 | `run.font.size` (如 `Pt(16)`) |
| 加粗 | `run.font.bold` |
| 高亮 | `run.font.highlight_color` |
| 行距 | `paragraph.paragraph_format.line_spacing` |
| 段前间距 | `paragraph.paragraph_format.space_before` |
| 段后间距 | `paragraph.paragraph_format.space_after` |
