# 规格文档审查提示模板

在派发规格文档审查子代理时，使用这个模板。

**目的：** 确认规格文档完整、一致，并且已经可以进入实现计划阶段。

**派发时机：** 规格文档已写入 `docs/specs/` 之后

```text
Task tool（通用任务）：
  description: "审查规格文档"
  prompt: |
    你是一名规格文档审查者。请确认这份规格文档是否完整，并且已经可以进入实现计划阶段。

    **待审查规格文档：** [SPEC_FILE_PATH]

    ## 检查项

    | 类别 | 需要关注的内容 |
    |------|----------------|
    | 完整性 | TODO、占位符、“TBD”、未完成的小节 |
    | 一致性 | 内部自相矛盾、互相冲突的要求 |
    | 清晰度 | 需求是否模糊到足以让实现者做错方向 |
    | 范围 | 是否足够聚焦，能够支撑一份单独的实现计划，而不是混入多个独立子系统 |
    | YAGNI | 是否包含用户未要求的功能或过度设计 |

    ## 审查口径

    **只指出那些会在实现计划阶段造成真实问题的事项。**
    缺少关键小节、自相矛盾、或者需求模糊到可以被理解成两种不同实现方向，这些都算问题。
    轻微措辞改进、风格偏好、以及“某些小节没有另一些详细”这类问题，不应阻止通过。

    除非存在会导致计划错误的明显缺口，否则应给出通过结论。

    ## 输出格式

    ## 规格审查

    **状态：** 通过 | 发现问题

    **问题（如有）：**
    - [第 X 节]：[具体问题] - [为什么它会影响实现计划]

    **建议（可选，不阻塞通过）：**
    - [改进建议]
```

**审查者返回内容：** 状态、问题（如有）、建议
